Learning Git and GitHub

 

Why learn Git and GitHub:

GitHub should have been named KitHub as it has all those K(c)ute 'K'ittens in its bootcamp. Git and GitHub are modern day arms and ammunitions for developers. Built on concepts of snapshots and not patches, it already has shots in it. Learning something new like Git and becoming code social thru GitHub is a great experience and a concrete step in embracing newer technology. They make complex tasks of application components' versioning, 'K'nitting collaborators' work and code sharing much simpler.

Our First Exercise:

As exercise, we created a situation of team working on a project, sharing and pushing components to a common repository on GitHub. We faced common scenarios of local clone out of sync with remote and components in conflict while pushing our changes thereby learning concept of pull, fetch and merge. Also how conflicts result in markers <<>> cropping into our files and then resolving those manually.

Branching - A Killer Concept:

Branching as introduced is truly a 'K'iller concept; developers day dreaming something different (a new idea), thinking of doing something differently but afraid of hurdles involved, can really try it out with branching in Git. This opens doors for new ideas to become innovations.

Learning Git and GitHub is definitely an ongoing task as we developers cruise thru our endeavors in this training and the assignments thereafter.
